The first attempts to create a baseline of reasonable defaults so that if every other aspect of your website fails, or is not supported by the browsing device, will display an attractive low-bandwidth mobile-friendly site. Responsive Web Design refers to the ability to have a layout adapt to a variety of screen sizes, resolutions, and types. Projects

Center for Civic Reflection

(Still in development) The Center for Civic Reflection (CCR) needed to replace an ageing custom built CMS that had been in use for almost 10 years. It needed to allow flexible data entry and search, and allow easy relationships to be defined between entries and entry-types. The application itself is, at it's core, a resource for civic-engagement facilitators to look up readings, discussion plans, facilitation summaries, and related media for use in targetted discussions and workshops. We built the site with ExpressionEngine due to it's ability to create custom data channels and views with ease.

Notable features include

  • rather than have CCR log into the control panel to manage the site (which can require extra training, and less control over functionality without tremendous development overhead), we streamlined the data-entry/management process by creating all the entry creation and management so it could be handled via the primary 'front-end' interface if logged in with a user account of sufficient privileges;
  • the ability to load forms-within-forms for entries which needed to be related to other entries which had not yet been added to the CMS;
  • wide range of related content displaying dynamically, depending on one's location in the site;

Worklocal

Worklocal is a job matching site which seeks to match potential applicants in the communities surrounding the new Green Exchange Business Park in Logan Square with tenants as they move into the building. The website needed to be able to provide application and resume forms for applicants, display current jobs in the system, and allow applicants to express interest in available positions.

The website also needed to be accessible by applicants both young and old, so the site had to function in older browsers that might be available in a library, or a computer which has not been frequently updated, as well as the latest smart-phones and mobile devices

Notable features include

  • responsive interface and layout which adapts from limited viewing environments such as phones and 7in tablets to traditional laptop and desktop screen sizes;
  • extensive reporting capabilities for registrants, applications, and resumes currently in the system;
  • applicant status tracking so worklocal team members can assist applicants in a targetted way to increase placement opportunities;
  • exporting of reports in spreadsheet consumable formats;

About this document

This resume is dynamic website built using HTML5, CSS3, and JQuery with html5boilerplate used for sensible defaults. It was designed with a Mobile First strategy, and Responsive Design techniques to adapt to larger devices and different device orientations. Codeigniter was used as the underlying development framework and provides the dynamic cover-letters and email obfuscation.

mPDF is the third party PHP library used to dynamically generate PDF versions of each resume. Enquire.js is used to dynamically load/unload JQuery functions depending on the active media-query.
